# 前后端分离介绍

本项目使用的是前后端分离搭建

  • 前端负责数据显示，单独的工程项目
  • 后端负责提供数据和操作数据，也是单独的工程项目
  • 前后端使用相同的数据格式进行交互，如json
  • 前端通过ajax调用后端暴露的api服务，传递json数据进行交互

好处：前后端解耦

  • 前后端定义好交互的数据格式之后，可以并行开发，相互不影响
  • 前端专注于页面数据展示以及用户体验，后端专注数据的存储以及服务端的性能
  • 在解决某些专属于前后端bug的情况下，前端服务改变，不需要重启后端，后端代码改变，也可以不影响前端

# 创建数据库与表

分别创建如下数据库

  • yygh-cmn 公用库
    • dict 字典表
  • yygh-hosp 医院库
    • hospital_set 医院元数据表
  • yygh-order 订单库
    • order_info 订单信息表
    • payment_info 支付信息表
    • refund_info 退款信息表
  • yygh-user 用户库
    • user_info 用户信息表
    • user_login_record 用户登录信息
    • patient 就诊人表

# 构建父模块 common

common 属于模块节点，内含

  • common-util
    • 工具类模块，所有模块都依赖它
  • service-util
    • service 服务的工具包，包含service服务的公共配置类，所有service模块都依赖于它

# 创建Maven模块

点击yygh_parent，右键选择New -> Module

选择maven 然后点击 Next

填写模块名称 common ，然后点击Finish

由于common属于父模块，也需要删除src目录
